# 课程设计要求

## 题目

**基于 Spring Boot 3 的校园流浪动物图鉴与动态打卡系统设计与实现**

## 1. 课程设计目的

本课程设计是计算机科学与技术/软件工程专业集中实践性教学环节。通过本课题的设计，要求学生：

1. 深入理解并掌握 Spring Boot 3.x 核心框架的设计思想与依赖注入机制。
2. 熟练运用 JDK 17/21 新特性、MyBatis-Plus（或 Spring Data JPA）持久层框架及 MySQL 数据库进行规范化开发。
3. 掌握基于 RESTful API 规范的后端接口设计方法、文件上传以及标准分层架构（Controller-Service-Mapper）。
4. 培养独立分析业务需求、设计关系型数据库表结构、解决多表关联查询及工程代码调优的实际动手能力。

## 2. 系统角色与权限定义

系统主要面向校园内的两类用户群体：

| 角色 | 职责 | 说明 |
|------|------|------|
| **普通学生 / 教工（User）** | 浏览全校流浪动物信息；偶遇动物时发布打卡动态 | 客户端用户 |
| **系统管理员（Admin）** | 校园流浪动物电子档案的建立、更新与日常维护 | 后台管理端 |

> 权限设计要求两类用户**功能边界清晰**：Admin 只管"图鉴"（增/删/改/查动物档案），User 只管"打卡"（浏览图鉴 + 发布打卡 + 查看时间轴）。

## 3. 系统核心功能模块与业务规范

系统核心围绕 **"动物档案"** 与 **"日常打卡"** 两条主线展开，**重点考核一对多（One-to-Many）的关系型业务构建**：

```
user ──┐
       ├──< check_in >── animal (admin managed)
       │                       ▲
       └──── 打卡 (Many) ──────┘
                              (One)
```

> 一个动物档案 → 多条打卡记录（`animal 1 : N check_in`）。动物详情页需要按 `animal_id` 关联查询并按时间倒序聚合。

### 3.1 校园动物图鉴管理模块（管理员端）

#### 档案录入

- 管理员录入新发现的流浪动物信息
- **字段**：
  - 暂定名字（`name`，字符串）
  - 动物类型（`type`，枚举：猫 / 狗）
  - 常驻区域（`area`，字符串，如：图书馆草坪、西区食堂）
  - 封面照片（`cover_image`，通过**本地文件上传**保存）
- 提交后生成一条动物档案

#### 档案检索

| 维度 | 要求 |
|------|------|
| 名字 | 支持**模糊查询**（LIKE） |
| 类型 | 支持按 猫 / 狗 条件**筛选** |
| 返回 | **分页**返回（page / size） |

#### 档案删除

- 支持对误填或无效档案进行删除
- 删除动物时需考虑**外键约束**（是否级联删除其下打卡记录？建议**保留打卡记录**，仅解除关联或软删动物）

### 3.2 偶遇近况打卡模块（用户端）

#### 即时打卡发布

- 学生在校园内偶遇某只动物时，可选择该动物的 `id`
- 在线编写一段文字描述（如："今天在教学楼下看到它了，喂了根猫条，精神状态良好"）
- 发布后**直接生成一条动态记录**
- 打卡记录字段（推断）：
  - `animal_id`（外键，关联动物档案）
  - `user_id`（外键，关联发布人）
  - `content`（文字描述）
  - `created_at`（发布时间）

#### 动态时间轴查询

- 用户点击某只动物的详情页时，后端根据 `animal_id` **关联查询**打卡动态表
- **按时间倒序**（最新发布的在最前）聚合返回该动物的历史足迹列表
- 在前端 / 接口端呈现为"时间轴近况"

## 4. 业务核心要点（提示）

- **一对多关系建模**：`animal (1) → check_in (N)`，是课程设计**核心考察点**
- **多表关联查询**：用户 + 打卡 + 动物 三表 JOIN，详情页可能需要展示"谁、在什么时间、看了哪只动物"
- **文件上传**：动物封面照片，需配置本地存储路径 + 静态资源映射
- **分页查询**：MyBatis-Plus 的 `Page<T>` 封装
- **RESTful 接口命名**：
  - `GET    /api/animals?name=&type=&page=&size=` — 档案检索
  - `POST   /api/animals` — 档案录入（multipart，含文件）
  - `DELETE /api/animals/{id}` — 档案删除
  - `POST   /api/check-ins` — 打卡发布
  - `GET    /api/animals/{id}/check-ins` — 时间轴
